## ข้อสังเกตเชิง robustness (เจอระหว่างรีวิว — แก้แล้วทั้งหมด)

1. **รูป item จาก API พัง (404)** — เพิ่ม `onError` fallback เป็นรูป default ตามประเภท (กัน loop ด้วย flag) + ใส่ guard `apiItem.type === itemType` ตอนส่ง prop กันแสดงชื่อ/ประเภท item ไม่ตรงเมื่อผู้ใช้เปลี่ยนประเภทเอง — **แก้แล้ว**
2. **`cancelRef` เป็น dead code** — ไม่มีจุดไหน set `true` (ปุ่มยกเลิกถูกถอดออกแล้ว) ลบทิ้ง — **แก้แล้ว**
3. **cooldown เดินต่อแม้พับแผง** — แยกพฤติกรรม: เปิดแผงใช้ `requestAnimationFrame` (ลื่น), พับแผงใช้ `setTimeout` ครั้งเดียว ไม่ re-render ทุกเฟรมตอนมองไม่เห็น — **แก้แล้ว**
